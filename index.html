Scheduling Algorithms

Process Scheduling is an OS task that schedules processes of different states like ready, waiting, and running. The main reason for using process scheduling is to keep the CPU busy at all times and get the minimum response time for programs.

Process Synchronization

Processes Synchronization is the way by which processes that share the same memory space are managed in by an OS. It helps maintain the consistency of data by using variables or hardware, that restrict the illegal activity.

Banker's Algorithm

The banker’s algorithm is a deadlock avoidance algorithm that tests for safety by simulating the allocation for predetermined maximum possible amounts of all resources before allocation.

Memory Management

Any process must be loaded onto the main memory before it can be executed by the CPU. Memory management is the task of allocating the memory among different processes effciently. Several algorithms exist for optimal allocation.

Page Replacement

In an operating system that uses paging for memory management, page replacement algorithms are needed to decide which page needs to be replaced whenever a new page comes in and there is insufficient main memory available.

Paging

Paging is a memory management scheme that eliminates the need for contiguous allocation of physical memory. This scheme permits the physical address space of any process to be non–contiguous.

Disk Scheduling

Disk scheduling is done by operating systems to schedule I/O requests arriving for the disk. Hard drives are one of the slowest parts of the computer system and hence must be accessed in an efficient manner.

File Allocation

The allocation methods define how the files are stored in the disk blocks. The main idea behind these methods is to provide efficient disk space utilization and fast access to the file blocks. The three main algorithms are discussed.
