Multi level paging is a paging technique which uses multiple age tables arranged in a hierarchical order to map the logical address space to the physical address space. If there are n levels in the scheme, then the entries in all the tables except the ones in the last level are pointers to page tables of the next level. The last level page tables consist of the actual frames in which the data is stored. The logical address generated consists of offsets for each level and the result obtained at each level is used to refer to a page table in the next level using this offset for that level.

Advantages:

  • Page number lookups are faster.
  • The page table can consume much less space if there are large regions of unused memory.
  • Each segment (code, data, stack) can be managed separately.
